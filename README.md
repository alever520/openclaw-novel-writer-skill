# Novel Writer Skill

OpenClaw 网络小说创作与管理的系统化工具。

---

## 功能特性

✅ **项目初始化**：一键创建规范化的小说项目结构  
✅ **章节创作**：基于大纲的系统化创作流程  
✅ **读者反馈管理**：记录、分析、处理读者意见  
✅ **最佳实践积累**：从参考资料中提取创作指南  
✅ **版本控制**：Git 版本管理，支持回滚  

---

## 快速开始

### 1. 初始化新小说项目

```
我想写一个异世界魔法小说，帮我初始化项目
```

Skill 会引导你：
- 创建项目目录结构 (chapters, references, docs)
- 初始化 Git 仓库并关联 GitHub
- 创建基础文件 (README, outline, writing-guide, reader-feedback)

### 2. 创作章节

```
写第一章
```

Skill 会自动：
- 读取大纲，确认剧情规划
- 读取创作指南和读者反馈
- 根据大纲创作章节内容
- 进行自检（剧情、语言、节奏）
- 提交到 GitHub

### 3. 处理读者反馈

```
有读者说第三章的实验过程太详细了，看着有点枯燥
```

Skill 会：
- 记录反馈到 reader-feedback.md
- 分析影响范围
- 提供处理建议
- 询问是否修改相关章节

### 4. 更新最佳实践

```
更新最佳实践
```

Skill 会：
- 检查 docs/ 文件夹的新增内容
- 提取关键要点
- 更新 writing-guide.md
- 应用到后续创作

## 核心功能说明

### 章节创作流程（v1.5.0）

**步骤1**: 读取 `references/outline.md`，掌握全局剧情，确认本章位置和任务  
**步骤2**: 创作章节大纲（七模块格式：核心任务/世界观锚点/场景规划/人物动向/节奏安排/伏笔处理/预计字数），向用户确认  
**步骤3**: 读取 `writing-guide.md` 和 `reader-feedback.md`，输出本章写作关键点（3-5条）  
**步骤4**: 按确认大纲创作章节正文，遵守步骤3总结的关键点  
**步骤5**: 自检清单（15项）  
**步骤6**: 提交到 GitHub  
**步骤7**: 章节总结与向用户汇报  

### 自检清单（步骤5）

创作完成后对照以下5大维度打勾，全部通过方可提交：

- **大纲执行**（4项）：场景序列、节奏、伏笔、字数
- **剧情**（3项）：符合全局大纲、推进主线、人物行为符合性格
- **语言**（4项）：无短句堆砌、流畅、无重复词、对话符合人设
- **节奏**（4项）：张弛有度、场景转换自然、爽点分布、信息密度适中
- **读者反馈**（3项）：待处理问题已处理、未重复已批评错误、采纳合理建议

### 读者反馈处理

1. **记录反馈**：日期、来源、章节、类型、内容
2. **分析影响**：影响章节、严重程度、是否需要修改
3. **提供建议**：立即修改 / 后续注意 / 忽略
4. **批量更新**：如需修改，逐章处理
5. **更新状态**：标记为"已处理"

### 版本管理

- **自动记录**：每次提交自动记录到 README.md
- **保留最近 5 次**：便于快速回溯
- **支持回滚**：可回滚到任意历史版本
- **安全措施**：回滚前创建备份分支

### 示例 1：启动新项目

**用户输入**:
```
初始化小说项目：《真理法则：浮空城编年史》
```

**Skill 输出**:
```
✅ 项目初始化完成！

【项目信息】
- 名称: 真理法则：浮空城编年史
- 路径: /root/.openclaw/workspace/truth-floating-city
- GitHub: https://github.com/yourusername/truth-floating-city-novel

【下一步】
1. 请完善大纲 (references/outline.md)
2. 如有参考资料，放入 docs/ 文件夹
3. 运行 "更新最佳实践" 整合参考资料
4. 准备好后，说 "开始写第一章"
```

## 最佳实践

1. **先完善大纲**：明确的大纲是创作的基础
2. **定期更新反馈**：及时记录和处理读者意见
3. **积累创作指南**：从参考资料中提取有用的原则
4. **频繁提交**：每章完成后立即提交，方便回溯
5. **版本备份**：重要修改前创建备份分支

---

## 注意事项

⚠️ **Git 认证**：首次使用需配置 GitHub Token  
⚠️ **自检辅助**：自检清单是辅助工具，最终质量需作者把关  
⚠️ **反馈选择**：并非所有反馈都需要处理，要有判断  
⚠️ **回滚谨慎**：强制推送会覆盖远程历史，务必确认
