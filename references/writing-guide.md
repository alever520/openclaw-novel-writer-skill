# 写作风格指南

## 语言风格

### ✅ 推荐做法

**自然流畅**
```markdown
❌ 避免：林墨深吸一口气。他闭上眼睛。他开始冥想。魔力涌动。
✅ 推荐：林墨深吸一口气，闭上眼睛开始冥想，魔力如潮水般涌动。
```

**口语化但不失文学性**
```markdown
❌ 避免：其乃吾之宿敌，必欲除之而后快。
✅ 推荐："这家伙，我迟早要收拾他。"林墨眯起眼睛。
```

**专业术语要解释**
```markdown
❌ 避免：他施展了"维度锚点·空间拓扑重构·四阶稳态矩阵"。
✅ 推荐：他施展"维度锚点"——一种稳定空间坐标的高阶法术。
```

### ❌ 避免的问题

1. **短句堆砌**（读起来像机器生成）
2. **过度使用"然而、但是、不过"**（节奏生硬）
3. **频繁重复相同句式**
4. **对话过长**（单次对话超过 100 字需要拆分）
5. **信息倾倒**（一口气解释 10 个设定）

## 章节节奏

### 单章结构（3000-5000 字）

**标准模板：**
1. **开场**（300-500 字）：接续上一章，或新场景开篇
2. **发展**（1500-2500 字）：核心事件展开
3. **高潮**（500-1000 字）：本章冲突点
4. **收尾**（200-500 字）：留悬念或过渡到下一章

### 重要情节给足篇幅

**战斗场景**：1500-2500 字
- 开始：双方对峙，氛围渲染
- 过程：招式拆解，策略博弈
- 结尾：胜负与影响

**情感戏**：1000-1500 字
- 避免"我爱你""我也爱你"式对话
- 通过细节描写（眼神、动作、环境）

**突破/升级**：1000-2000 字
- 突破前的困境
- 领悟过程（不要一句话带过）
- 突破后的变化（能力、心态）

### 场景转换

**自然过渡**
```markdown
✅ 空行 + 时间/地点提示：

三天后，真理之塔。

"导师，实验结果出来了。"艾莉娅推门而入。
```

**避免生硬**
```markdown
❌ 【场景一：实验室】
❌ 第一幕：深夜的图书馆
❌ ——分割线——
```

## 对话写作

### 简洁有力

```markdown
❌ "我认为，基于我们目前掌握的情报和资源，以及对方的实力评估，我们应该..."
✅ "敌人太强，正面硬刚必死。"林墨斩钉截铁。
```

### 符合人设

**学者型角色**：
```markdown
"根据能量守恒定律，这个法阵的输出不可能超过输入。除非..."
```

**战士型角色**：
```markdown
"少废话！干就完了！"
```

**腹黑型角色**：
```markdown
"哦？你确定要这么做吗？"他笑而不语，眼神意味深长。
```

### 对话与描写交织

```markdown
❌ 纯对话：
"你来了。"
"是的。"
"坐吧。"
"好。"

✅ 交织：
"你来了。"导师放下羽毛笔,抬起头。
"是的。"林墨在对面坐下,注意到桌上摆满了古老的卷轴。
```

## 爽点设计

### 小爽点（2-3 章一次）

**类型**：
- 学术突破（改良法术成功）
- 小规模战斗胜利（击败小怪）
- 获得新物品/技能
- 打脸/反转（对手被啪啪打脸）
- 意外收获（本来失败,结果发现新东西）

**示例**：
```markdown
"不可能！这个法术效率怎么会这么高？！"
导师盯着数据,震惊到说不出话。

林墨淡然一笑："只是换了个思路而已。"
```

### 中爽点（10-15 章一次）

**类型**：
- 晋升/突破（三环 → 四环法师）
- 获得重要资源（法师塔、稀有材料）
- 击败强敌（与同级对手的决战）
- 名声大噪（发表论文震动学术界）

**示例**：
```markdown
魔力洪流涌入体内,
林墨感觉自己的灵魂都在升华。
四环了。
仅仅两年,他就走完了别人十年的路。

睁开眼,世界变得不同了。
```

### 大爽点（卷末）

**类型**：
- 击败本卷 BOSS
- 建成浮空城等宏伟工程
- 神战胜利
- 位面征服

**要求**：
- 篇幅充足（2000+ 字）
- 多方势力见证
- 影响深远（改变世界格局）
- 情绪饱满（震撼、热血、感动）

## 常见问题与解决

### 问题 1：信息量过载

**症状**：一章塞入 10 个新设定
**解决**：
- 分散到多章
- 通过对话自然引出
- 用"初次提及+后续深入"模式

### 问题 2：节奏拖沓

**症状**：主角在同一地点待了 5 章没有进展
**解决**：
- 检查大纲,确认关键情节点
- 加快推进（时间跳跃、事件催化）
- 增加外部压力（敌人出现、时间限制）

### 问题 3：角色扁平

**症状**：配角只是工具人
**解决**：
- 给配角独立目标
- 设计角色之间的冲突
- 展示角色成长

### 问题 4：战斗无聊

**症状**："A 出招,B 躲开,A 再出招..."
**解决**：
- 加入策略博弈（诱敌、佯攻）
- 环境互动（利用地形、天气）
- 心理描写（紧张、计算、领悟）

## 检查清单

创作完一章后,对照检查:

- [ ] 章节长度在 3000-5000 字
- [ ] 有明确的开始、发展、高潮、收尾
- [ ] 对话简洁且符合人设
- [ ] 没有短句堆砌
- [ ] 场景转换自然
- [ ] 有至少一个"爽点"或"钩子"
- [ ] 与大纲设定一致
- [ ] 与前文内容连贯

## 风格参考

### 理性流（科学魔法向）
- 重视逻辑推理
- 详细的原理解释（但不要写成论文）
- 主角靠智慧而非运气

### 热血流（战斗升级向）
- 快节奏
- 战斗描写详细
- 情绪渲染到位

### 权谋流（政治博弈向）
- 多方势力交锋
- 阴谋与反阴谋
- 信息不对称

### 日常流（治愈轻松向）
- 温馨日常
- 角色互动
- 慢节奏但有趣

---

**使用建议**：
- 创作前读一遍对应类型的要点
- 创作后用检查清单验收
- 遇到问题时查阅"常见问题"

## 章节大纲格式规范

每章动笔前，必须完成七模块章节大纲并经作者确认：

```
📋 第XXX章《[标题]》章节大纲

**核心任务**
[具体到"谁在什么压力下做了什么"，1-2句，不写"推进剧情"这类泛指]

**世界观锚点**（嵌入正文，不旁白堆砌）
- [设定名称]：[一句话说明 + 对本章剧情的具体作用]

**场景规划**

场景一：[场景名]（[地点，时间]）
[3-5句：发生了什么 + 关键对话示例（至少1句）+ 场景结束时人物/情绪变化]

场景二：[...]

**人物动向**
- [角色名]：[起点 → 终点，行动逻辑]

**节奏安排**
- 开篇钩子：[第一句话/动作，为什么抓人]
- 小爽点：[位置+内容]
- 结尾留钩：[悬念或情绪余韵]

**伏笔处理**
- 新埋：[具体内容 / 无]
- 回收：[具体内容 / 无]
- 延续：[上章遗留 / 无]

**预计字数**：约XXXX字
```

**关键原则**：场景内对话示例为必填项，锁定人物在该时刻的状态，防止创作时写偏。

## 信息传递原则

**读者跟着人物的行为和心理活动来了解信息，不跟着作者铺垫。**

- ✅ 人物起身、看到某物、感受到某事 → 信息自然带出
- ✅ 内心独白处理信息时顺带交代背景
- ❌ 大段环境扫描后再切入人物行动
- ❌ 先交代完所有背景再让人物开口

判断标准：删掉这段，剧情推进是否受阻？不受阻 → 可能是多余铺垫。
